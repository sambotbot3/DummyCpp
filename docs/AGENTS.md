# AGENTS.md - Documentation

Scope: project documentation under `docs/`.

- Keep durable project notes here rather than in the root README.
- Put unresolved, condensed questions only in `questions.md`.
- Move resolved questions into `decisions.md` when a decision is made.
- Keep architecture, roadmap, plans, and resources separated by purpose.
- Prefer concise updates that point to code, examples, or tests when possible.
- Avoid code snippets in documentation unless they are essential; prefer prose and file/script references.
